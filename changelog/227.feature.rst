`~stixpy.product.sources.cal.find_energy_calibration_file_for_time` now requires the selected file's ``LIVETIME`` (primary FITS header) to be above a configurable threshold (default ``30 000 s`` ≈ 8 h). The search window has been extended from ±1 day to ``[-window_past, +window_future]`` with defaults of 10 days back / 1 day forward, so older calibration files can be used when the most recent ones fall below the threshold. Candidates are scanned via HTTP range requests (`astropy.io.fits.open(url, use_fsspec=True)`) so only the winning file is fully downloaded. All three new parameters (``min_livetime``, ``window_past``, ``window_future``) are keyword-only and overridable per call.
